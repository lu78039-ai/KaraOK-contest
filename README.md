# 🎤 專業卡拉 OK 大賽評分與管理系統

這是一個基於 Google Apps Script (GAS) 與現代 Web 技術開發的專業卡拉 OK 大賽管理系統。專為校園、社團或中大型比賽設計，提供從報名、抽籤、評分到成績公佈的完整數位化流程。

## 🌟 核心功能

1.  **獨立報名系統 (`register.html`)**
    *   **學生專屬介面**：與後台物理隔離，確保系統安全。
    *   **重複偵測**：自動比對班級、座號、姓名，防止重複報名。
    *   **二次驗證修改**：報名後提供序號，需憑序號與組員姓名方可修改資料。
    *   **截止控制**：可於 Google 試算表設定截止日期與時間（精確至分）。

2.  **主控門戶 (`index.html`)**
    *   **評審評分**：iPad 優化的流暢評分介面，支援即時送出。
    *   **成績看板**：大螢幕專用即時排行榜，支援「隱藏/顯示名次」的懸疑感設計。
    *   **管理系統**：提供隨機抽籤、出場序手動調整（支援拖曳感按鈕）、報名資料校正。
    *   **總分明細表**：一鍵生成包含所有評審明細的 A4 列印報表。

3.  **自動化統計後台 (`Code.gs`)**
    *   **即時彙整**：自動計算各隊平均分數與排名。
    *   **鎖定機制**：使用 `LockService` 解決多評審同時送出的衝突。
    *   **靈活設定**：可自由增減評審人數、評分項目及其權重。

## 📊 Google Sheets 資料庫架構

請建立一份試算表，並包含以下工作表：

### 1. `Students` (報名資料)
*   **欄位**：`報名序號` | `出場序` | `班級` | `座號` | `姓名` | `隊伍名稱` | `曲目` | `報名時間` | `影片連結`

### 2. `Judges` (評審名單)
*   **欄位**：`評審姓名` | `密碼`

### 3. `Scores` (評分紀錄)
*   **欄位**：`時間戳記` | `評審名稱` | `隊伍名稱` | `[評分項目1]` | `[評分項目2]` ... | `總分`

### 4. `Config` (系統設定)
*   **欄位**：`設定項目` | `值`
*   **項目包含**：`報名截止時間` (格式: 2026/5/20 23:59)、`顯示 4 名後分數` (TRUE/FALSE)

## 🚀 部署指南

1.  **部署 GAS 後端**：
    *   在 Google 試算表中點選「擴充功能」>「Apps Script」。
    *   貼入 `Code.gs` 並「部署」為「網頁應用程式」。
    *   設定存取權限為「所有人」。
2.  **設定前端連線**：
    *   將產生的 URL 填入 `config.js` 中的 `GAS_API_URL`。
3.  **上傳至 GitHub**：
    *   將所有檔案上傳至您的儲存庫。
    *   開啟 GitHub Pages 功能，獲取正式網址。

## 📂 檔案清單說明
*   `management.html`: 工作人員與評審的主入口 (隱藏網址，請妥善保管)。
*   `index.html`: 學生報名與資料修改的門戶頁面 (根目錄預設路徑)。
*   `report.html`: 成績列印專用頁面。
*   `style.css`: 統一的現代化深色 UI 樣式表。
*   `config.js`: 全域配置。
*   `judge.js` / `dashboard.js` / `admin.js` / `report.js` / `register.js`: 各模組的邏輯處理。
