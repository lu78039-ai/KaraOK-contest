# 系統架構規劃與實作指南

## 一、 問題與需求分析
*   **問題**：傳統紙本評分表統計耗時、容易計算錯誤，且無法即時呈現結果。
*   **原因**：缺乏數位化輸入介面與自動化統計後台。
*   **解法**：建立基於 Google Apps Script (GAS) 的前後端分離系統。使用 iPad 作為前端評分介面，Google Sheets 作為後端資料庫，實現即時送出與運算。
*   **風險**：多位評審同時送出成績可能導致資料庫寫入衝突（Race Condition），必須導入 `LockService` 進行防護。

## 二、 Google Sheets 資料庫建構規範
請開啟一份新的 Google 試算表，並建立以下三個工作表（名稱必須完全一致）：

### 1. 工作表名稱：`Students`
*   **用途**：儲存學生與隊伍基本資料。
*   **欄位（第一列）**：`班級` | `座號` | `姓名` | `隊伍名稱`
*   **注意**：前端系統會自動抓取「隊伍名稱」作為評分目標。

### 2. 工作表名稱：`Settings`
*   **用途**：控制評分項目與計算權重。
*   **欄位（第一列）**：`項目` | `權重`
*   **資料範例**：
    *   唱功 | 1.0
    *   演奏 | 0.8
    *   台風 | 1.2
    *   整體表現 | 1.0

### 3. 工作表名稱：`Scores`
*   **用途**：自動紀錄來自前端的每一筆評分。
*   **欄位（第一列）**：`時間戳記` | `評審名稱` | `隊伍名稱` | `唱功` | `演奏` | `台風` | `整體表現` | `總分`

## 三、 部署流程拆解
1.  **部署 GAS 後端**：
    *   在 Google 試算表中點選「擴充功能」>「Apps Script」。
    *   將已建立的 `Code.gs` 內容複製貼上。
    *   點選「部署」>「新增部署作業」。
    *   類型選擇「網頁應用程式 (Web App)」。
    *   執行身分選擇「我 (Me)」，存取權限選擇「所有人 (Anyone)」。
    *   複製部署後產生的 `網頁應用程式網址 (URL)`。
2.  **設定前端連線**：
    *   開啟專案資料夾中的 `config.js`。
    *   將上一步複製的網址貼入 `GAS_API_URL` 變數中。
3.  **開始使用**：
    *   直接在瀏覽器或 iPad 上開啟 `index.html` 即可進入系統。

## 五、 GitHub 部署指南
1.  **建立儲存庫**：在 GitHub 上新增一個 Repository。
2.  **上傳程式碼**：將本地端的所有檔案（`.html`, `.js`, `.css`, `Code.gs`）推送到 GitHub。
3.  **啟用 GitHub Pages**：
    *   在 GitHub Repository 設定中找到「Settings」>「Pages」。
    *   Source 選擇 `Deploy from a branch`，Branch 選擇 `main` (或 `master`)。
    *   儲存後，您將獲得一個專屬的網址（例如：`https://username.github.io/repo-name/`）。
4.  **跨網域設定 (CORS)**：
    *   由於本系統採用前後端分離，GAS 後端預設已支援 GitHub Pages 的跨網域請求，無需額外設定。
    *   請確保 `config.js` 中的 `GAS_API_URL` 為最新的部署網址。

## 六、 檔案清單說明
*   `index.html`: 系統入口首頁。
*   `judge.html/js`: 評審專用評分介面。
*   `dashboard.html/js`: 即時大螢幕排行榜。
*   `report.html/js`: 列印用的結算總表。
*   `admin.html/js`: 報名隊伍管理與出場序抽籤系統。
*   `Code.gs`: 需部署於 Google 試算表後端的 Apps Script 原始碼。
*   `style.css`: 統一的視覺介面樣式表。
*   `config.js`: 全域設定檔（儲存 API 網址）。
